# Mission Control 使用说明

## 🌐 仪表盘地址

**https://darren009.github.io/mission-control-dashboard/**

（建议收藏到浏览器书签）

## 📝 如何创建任务

1. 打开仪表盘网址
2. 点击右上角"+ New Task"
3. 填写任务信息：
   - **Title**（标题）：任务名称
   - **Description**（描述）：详细说明，告诉灵宝要做什么
   - **Subtasks**（子任务）：可选，把大任务拆成小步骤
   - **DoD**（Definition of Done，完成标准）：可选，怎样算完成
   - **Priority**：high / medium / low
4. 点击"Save"

新任务会出现在"Backlog"列。

## 🚀 如何启动任务

**方法 1：拖拽卡片**
- 把任务卡片从"Backlog"拖到"In Progress"列
- 松开鼠标后，系统会自动保存

**方法 2：点击任务卡片**
- 点击任务卡片打开详情
- 修改"Status"为"In Progress"
- 点击"Save"

灵宝会在 **1 分钟内** 收到通知并开始干活。

## 📊 任务状态说明

| 状态 | 说明 | 谁操作 |
|------|------|--------|
| **Backlog** | 待办，还没开始 | 你创建任务 |
| **In Progress** | 进行中，灵宝在干活 | 你拖过去 → 灵宝开始 |
| **Review** | 等待审核，灵宝做完了 | 灵宝完成后移到这里 |
| **Done** | 已完成并批准 | 你审核通过 |

## 🔄 如果需要返工

1. 灵宝做完后会把任务移到"Review"
2. 你检查结果，如果不满意：
3. 在任务卡片的评论里写反馈
4. 把任务拖回"In Progress"
5. 灵宝会看到反馈并重新处理

## 📱 查看进度

任务卡片上会显示：
- 子任务完成情况（2/5 表示完成了 2 个，还有 3 个）
- 最近的评论
- 优先级标签（红/黄/绿）

## ⚙️ 技术说明

**工作原理**：
- 你在网页上操作 → 自动保存到 GitHub
- 灵宝每分钟检查一次 GitHub 数据
- 发现"In Progress"状态 → 通知灵宝开始干活
- 灵宝完成后会更新任务状态并 push 到 GitHub

**数据存储**：
- 所有数据存在 GitHub 仓库：https://github.com/darren009/mission-control-dashboard
- 如果需要备份/迁移，直接 clone 仓库即可

**延迟说明**：
- 你拖任务后，灵宝会在 **1 分钟内** 收到通知
- 对于耗时几分钟以上的任务，这个延迟可以忽略

## 🐛 常见问题

**Q: 拖动任务后看不到变化？**
A: 刷新浏览器页面（F5）

**Q: 灵宝没有反应？**
A: 
1. 检查任务是否真的在"In Progress"列
2. 等待 1-2 分钟（检查周期）
3. 如果还没反应，直接发消息给灵宝

**Q: 任务卡片乱了？**
A: 刷新页面，GitHub 是唯一真实数据源

## 💡 最佳实践

1. **任务描述要清楚**：灵宝需要知道具体要做什么
2. **善用子任务**：把复杂任务拆成小步骤
3. **设置完成标准**：在 DoD 里写清楚"怎样算做完"
4. **及时反馈**：灵宝做完后尽快审核，需要改的话写清楚要改什么
